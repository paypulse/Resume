01 자기소개서 Cover Letter

Spring Boot 기반 백엔드 시스템 설계와 운영에 강점을 가진 7년차 개발자입니다.

수동으로 처리되던 반복 업무를 자동화할 때 가장 큰 보람을 느낍니다. 복잡한 데이터 처리 로직을 API로 분리하고, Jenkins 기반 CI/CD 파이프라인과 Spring Batch로 운영 효율을 높인 경험처럼, 사람이 직접 해야 했던 일을 시스템이 대신하도록 만드는 것을 즐깁니다.

🔧 기술적 강점

JPA/QueryDSL을 활용한 복잡한 데이터 처리, AWS 인프라 구축 및 CI/CD 파이프라인 구성, 멀티 모듈 아키텍처 설계 경험을 보유하고 있습니다. IoT FOTA 시스템, Drools 룰 엔진, YOLO 객체 탐지 연동 등 낯선 기술도 직접 파고들어 실서비스에 적용해왔습니다.

🤝 일하는 방식

팀의 마무리 역할을 자처하며 놓치기 쉬운 부분까지 챙기는 스타일입니다. 문서화되지 않은 레거시 코드도 직접 분석하고, 단기 운영 요구사항과 장기 확장성을 함께 고려한 개발을 지향합니다.

🚀 앞으로의 방향

성장하는 환경에서 제 역량을 충분히 발휘하고 싶습니다. 기술적 도전이 있는 곳에서 팀과 함께 문제를 해결하며, 서비스의 핵심 백엔드를 설계하고 운영하는 개발자로 성장하고 싶습니다.

02 기술 포트폴리오 Technical Portfolio

🔄 FOTA 업데이트 요청 및 배포 흐름

링스쿨 FOTA · 2024.03–2024.12

대규모 디바이스를 대상으로 원격 펌웨어 업데이트를 수행하는 시스템. 관리자가 업데이트를 요청하면 디바이스별 Task를 생성하고, MQTT를 통해 명령을 전송하며 실시간으로 진행 상태를 동기화합니다.

정상 배포 흐름 관리자 업데이트 요청 → 검증 → 디바이스 목록 조회 → 상태 잠금 → Task 생성 + MQTT Publish → 다운로드 + 설치 + 재부팅·검증 → 완료 이벤트 수신 → 배포 완료 ⚠ 핵심 난제: 동시 업데이트 트래픽 제어 / 상태 정합성 유지 / 부분 실패 허용 설계 / 트랜잭션 분리
실패 재처리 및 복구 흐름 Scheduler 실패 감지 → 미응답 디바이스 조회 → 재시도 횟수 판단
├ [가능] 카운트++ → 명령 재전송 → 성공 처리
└ [초과] 실패 기록 → 롤백 지정 → 운영자 알림 → 리포트 ⚠ 핵심 난제: 재시도 정책 설계 / 실패 유형 분류 / 장기 미응답 처리 / 운영 안정성 확보
